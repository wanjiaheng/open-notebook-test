# 需求文档

> **项目**：Open Notebook 多租户用户管理扩展  
> **版本**：v1.0  
> **日期**：2026-03-15  

---

## 1. 背景与目标

Open Notebook 是一款开源 AI 研究助手，原版本不含用户鉴权与数据隔离机制。本次扩展目标：

- 增加用户登录/注册与管理员审批流程
- 实现基于组织的多租户数据隔离
- 支持细粒度的组内管理员权限

---

## 2. 用户角色定义


| 角色    | 标识                           | 说明                    |
| ----- | ---------------------------- | --------------------- |
| 超级管理员 | `role = admin`               | 拥有全系统权限，可管理所有用户、组织、数据 |
| 组内管理员 | `member_of.role = org_admin` | 仅可管理所属组织内的用户和数据       |
| 普通用户  | `role = user`                | 只能查看/操作自己及所属组织的数据     |


---

## 3. 功能需求

### 3.1 用户认证


| 编号      | 需求                                         | 优先级 |
| ------- | ------------------------------------------ | --- |
| AUTH-01 | 用户通过邮箱/密码登录，获取 JWT Token                   | 高   |
| AUTH-02 | 新用户注册后状态为 `pending`，需管理员审批后才能登录            | 高   |
| AUTH-03 | 管理员可在后台直接创建用户并设为激活状态                       | 高   |
| AUTH-04 | 支持账号封禁（`suspended`）和恢复激活                   | 中   |
| AUTH-05 | 用户无法对自己执行封禁/删除操作                           | 高   |
| AUTH-06 | JWT 中携带 `sub`（用户ID）、`role`、`admin_org_ids` | 高   |


### 3.2 用户管理（管理后台）


| 编号      | 需求                                 | 优先级 |
| ------- | ---------------------------------- | --- |
| USER-01 | 超级管理员可查看所有用户，组内管理员仅查看所管理组织的用户      | 高   |
| USER-02 | 超级管理员可创建任意角色用户（普通用户/管理员），不可创建超级管理员 | 高   |
| USER-03 | 组内管理员创建用户时，只能分配自己有管理权限的组织          | 高   |
| USER-04 | 管理员审批用户激活时，自动将用户加入「公开」组织           | 中   |
| USER-05 | 组内管理员审批用户时，自动将用户加入该管理员所管理的所有组织     | 中   |
| USER-06 | 用户管理页面支持按组织、用户名、邮箱、状态筛选            | 中   |
| USER-07 | 用户侧边栏显示所属组织（超级管理员显示「全组织权限」）        | 低   |


### 3.3 组织管理


| 编号     | 需求                         | 优先级 |
| ------ | -------------------------- | --- |
| ORG-01 | 超级管理员可创建/编辑/删除组织           | 高   |
| ORG-02 | 系统自动初始化「公开」组织，不可删除、不可重命名   | 高   |
| ORG-03 | 「公开」组织对所有已登录用户可见           | 高   |
| ORG-04 | 用户可关联多个组织，每个组织中可设置独立的组内管理员 | 高   |
| ORG-05 | 「公开」组织不可通过 UI 手动添加/移除用户    | 中   |
| ORG-06 | 组织管理菜单仅超级管理员可见             | 高   |


### 3.4 笔记本（Notebook）数据隔离


| 编号    | 需求                              | 优先级 |
| ----- | ------------------------------- | --- |
| NB-01 | 笔记本可关联多个组织（`org_ids` 数组），默认个人可见 | 高   |
| NB-02 | 关联「公开」组织后，所有已登录用户均可查看           | 高   |
| NB-03 | 关联「公开」与其他组互斥（选公开则覆盖所有组）         | 中   |
| NB-04 | 笔记本列表页支持按组织筛选                   | 中   |
| NB-05 | 笔记本卡片显示创建者姓名                    | 低   |
| NB-06 | 超级管理员可查看所有用户的笔记本                | 高   |


### 3.5 来源（Source）数据隔离


| 编号     | 需求                           | 优先级 |
| ------ | ---------------------------- | --- |
| SRC-01 | 来源页面仅显示当前用户可访问的来源            | 高   |
| SRC-02 | 来源的可见性与关联笔记本的组织一致（通过笔记本引用推导） | 高   |
| SRC-03 | 公开笔记本的来源，所有已登录用户均可查看         | 高   |
| SRC-04 | 来源处理状态消息使用中文                 | 低   |


### 3.6 向量搜索数据隔离


| 编号        | 需求                   | 优先级 |
| --------- | -------------------- | --- |
| SEARCH-01 | 向量搜索结果仅返回当前用户有权访问的数据 | 高   |


### 3.7 UI/UX


| 编号    | 需求                        | 优先级 |
| ----- | ------------------------- | --- |
| UI-01 | 管理后台布局为左侧菜单 + 右侧内容区       | 中   |
| UI-02 | 侧边栏左下角显示用户名和所属组织，悬停展示详细信息 | 中   |
| UI-03 | 笔记本详情页有返回按钮               | 低   |
| UI-04 | 下拉菜单触发按钮（三个点）点击后不消失，可反复触发 | 中   |
| UI-05 | 弹框关闭后，卡片点击事件正常恢复          | 高   |


---

## 4. 非功能需求

- **安全性**：密码 bcrypt 哈希存储；JWT 签名验证；未认证请求拦截
- **性能**：分页查询；组织过滤基于数据库索引
- **可维护性**：数据库变更通过版本化迁移脚本管理
- **兼容性**：前端支持简体中文界面（i18n）

---

## 5. 约束与边界

- 超级管理员不能通过 UI 被创建（仅系统初始化或直接数据库操作）
- 「公开」组织名称固定为 `公开`，不得修改
- 用户不能自己封禁/删除自己
- 来源的组织权限通过所关联笔记本推导，来源本身仍保留 `user_id`/`org_id` 字段
